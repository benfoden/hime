<objective>
Harden the provider layer — the single place where the extension talks to the
network. Add a 10s AbortController timeout (ERR-03), replace the generic error
throw with classifyError so failures carry a typed kind and actionable message
(ERR-04/05, LOG-02), sanitize output with stripWrappers (FORM-08), and tune the
Auto-formality instruction so register inference works (FORM-06/07). The two
provider files currently duplicate the prompt builder verbatim; consolidate into
one shared module so the tuned instruction cannot drift between providers.

Purpose: This is the root-cause layer for the Phase 2 gaps — no timeout exists
today (silent hangs), errors collapse to one generic string, output is returned
raw, and the Auto instruction is too vague to reliably steer register.

Output: src/providers/prompt.ts (new shared builder), hardened openai.ts and
gemini.ts, expanded unit tests for the prompt builder, and a human-verify
checkpoint for live register inference.
</objective>

| provider API -> extension | Untrusted response body parsed for error message and translation content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Information disclosure | Gemini API key in URL query string | accept | Pre-existing Phase 1 design (Gemini API requires key as query param); key already only sent over HTTPS to Google. classifyError receives only status + body message, never the key |
| T-02-05 | Denial of service | hung provider fetch | mitigate | 10s AbortController timeout guarantees the request cannot hang indefinitely (ERR-03 root cause) |
| T-02-06 | Information disclosure | error message built from response body | mitigate | classifyError interpolates only provider `bodyMessage`; API key is never passed to classifyError so it cannot appear in a surfaced message or log |
</threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| provider API → extension | Untrusted JSON response body parsed into error messages and translation output |
| LLM output → host page field | Model-generated text inserted into the user's focused input via execCommand |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information disclosure | classifyError message built from `response.bodyMessage` | mitigate | Only include provider-supplied `bodyMessage` text in the 'unknown' message; never interpolate the API key (key lives only in fetch headers, never passed to classifyError) |
| T-02-02 | Tampering | stripWrappers on LLM output | accept | Output is inserted as plain text via execCommand insertText (not innerHTML); no HTML/script execution path. stripWrappers is cosmetic, not a security control |
| T-02-03 | Denial of service | unbounded LLM response length | accept | BYOK; user controls their own model/cost. Output bounded by provider max tokens; no extension-side amplification |
</threat_model>

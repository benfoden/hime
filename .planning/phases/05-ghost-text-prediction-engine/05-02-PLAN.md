<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page DOM ↔ content script | Ghost overlay/span injected into an arbitrary, possibly-hostile page |
| page input → background | Text-before-cursor read from the focused field crosses to background/provider |
| background → page DOM | Sanitized suggestion text is written into a DOM node visible on the page |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Information disclosure | ghost overlay readable by page scripts | mitigate | Overlay/span is `pointer-events: none`, carries no user-secret beyond text already sent to the provider, and is removed on blur/focusout; suggestion text is the model output, not field secrets |
| T-05-06 | Information disclosure | sampling ineligible fields | mitigate | `isValidInputElement` (reused verbatim) rejects password/readonly/hidden/disabled + canvas editors before any text is read or sent (LANG-01 / PRED-05 data-exfil guard) |
| T-05-07 | Tampering | suggestion text injection into DOM | mitigate | Ghost rendered via `textContent` only (never innerHTML); `sanitizeGhost` strips control chars/newlines before write — model output cannot break the overlay or inject markup |
| T-05-08 | Tampering | committed text corruption | mitigate | Ghost is overlay/non-editable-span only — never assigns `element.value`/`innerText`; accept uses `execCommand('insertText')` at caret (undo-safe), preserving native content + undo stack |
| T-05-09 | Denial of service | stale response renders into wrong field | mitigate | seq + element guard (`seq !== currentRequestSeq || element !== currentElement`) plus AbortController abort on supersession and on blur |
</threat_model>

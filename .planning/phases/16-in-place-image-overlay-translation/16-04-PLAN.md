<objective>
Wire the content-script overlay layer that turns the Plan 02 worker replies into anchored, legible, swappable on-image overlays. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page DOM (untrusted) → content script | Page-supplied `<img>` elements and their src are untrusted; only src is read + sent to the worker. |
| OCR/translated text → overlay DOM | Untrusted text rendered into overlay boxes — must be textContent, never innerHTML. |
| content → worker (translateImageBlocks) | Message carries srcUrl/dedupKey/config only — no key. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-09 | Tampering/Elevation (XSS) | overlay box + toggle text | mitigate | All box/button/label text set via `textContent` ONLY — never innerHTML (repo-wide law, content.ts:1132,1733). OCR/LLM text is untrusted. |
| T-16-02 | Information Disclosure | translateImageBlocks payload | mitigate | Payload carries `{srcUrl, dedupKey, config}` only; the Plan 01 message type has no key field, so a key cannot be serialized from the content script. |
| T-16-10 | Information Disclosure (tainted canvas) | fitText measureText canvas | mitigate | The throwaway 2D canvas used for measureText draws NO image — only text is measured — so no cross-origin image is ever read (no taint; no-inpainting lock honored). |
| T-16-06 | Tampering (mis-placed geometry) | capture-fallback images | mitigate | `reply.captureFallback` images are skipped — no overlay rendered for unmappable screenshot geometry (Pitfall 2). |
| T-16-11 | Denial of Service | unbounded image pass | mitigate | The pass reuses progCreateBudget + progCreateConcurrencyGate (phase-13 cost gates, D-01) so the number of in-flight OCR/translate jobs is bounded. |
| T-16-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs (milestone guardrail: zero new OSS deps). |

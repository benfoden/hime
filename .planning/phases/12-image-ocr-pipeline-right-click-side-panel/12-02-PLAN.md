<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker → Google REST | BYOK key crosses to vision.googleapis.com / translation.googleapis.com over TLS via ?key=; must never be logged or interpolated into error strings |
| Google response → ImageResult | untrusted provider text enters the result object (rendered textContent-only downstream by Plan 04) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03 | Information disclosure | apiKey in vision-google.ts | mitigate | Key set via `URL.searchParams.set('key', apiKey)` (auto-encoded), never string-interpolated, never logged; Task 1 case 5 asserts the key is absent from thrown error messages/stacks (brave-search.ts:9-11 doctrine). |
| T-12-04 | Spoofing/Tampering | Google error responses | mitigate | `!response.ok` routed through `classifyError('google', null, {status, bodyMessage})`; only the classified message (not raw body) is surfaced; no key echoed. |
| T-12-05 | Denial of service | two-call sequence vs MV3 30s fetch-stall | mitigate | Per-call AbortController timeout (~12s each) so neither call hangs the worker; timeout maps to a network-kind error, not a silent hang (A5). |
</threat_model>

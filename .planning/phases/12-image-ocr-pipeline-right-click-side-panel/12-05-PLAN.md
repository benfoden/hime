<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page (content script) ŌåÆ worker | the right-click context provides only `info.srcUrl` + `tab`; the key is NEVER passed from the page |
| worker ŌåÆ arbitrary image origin | SW fetches arbitrary `<img>` bytes under `<all_urls>` (SSRF-ish surface) |
| worker ŌåÆ Google REST | BYOK key crosses to Google; stays worker-side |
| worker globals Ōåö storage.session | MV3 termination wipes globals; durable state must survive |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-11 | Information disclosure | googleApiKey | mitigate | Read via `settings.googleApiKey` in the worker only; auth-error if empty; never accepted from the payload; never logged (searchTranslated:176-181 + brave-search doctrine). [IMG-07, ASVS V4] |
| T-12-12 | Elevation of privilege / SSRF | SW `fetch(srcUrl)` of arbitrary URLs under `<all_urls>` | mitigate | Only `info.srcUrl` from an explicit user right-click on an `<img>` is fetched (not arbitrary worker input); response is MIME-guarded + size-capped before send; no redirect-following of attacker-chosen internal hosts is added. `<all_urls>` breadth documented (already in content_scripts.matches ŌåÆ unchanged install prompt). |
| T-12-13 | Tampering | tainted-canvas pixel read in page | mitigate | Pixels resolved in the SW (fetch) or via `captureVisibleTab` ŌĆö never via a tainted canvas in the content script (Pitfall 2). |
| T-12-14 | Denial of service | MV3 worker sleeps mid-job ŌåÆ panel hangs | mitigate | Job/dedup/result in `storage.session`; ~25s `Promise.race` timeout ŌåÆ typed error entry; panel rebuilds from storage.session on open (Pitfall 5). |
| T-12-15 | Denial of service | oversized image ŌåÆ opaque 400 | mitigate | `downscaleAndGuard` enforces VIS-03 long-edge/MIME limits before send. |
| T-12-SC | Tampering | npm/pip/cargo installs | mitigate | No runtime installs this plan; the only dep change (the @types/chrome bump) is in Plan 01 and is a first-party dev bump (RESEARCH Package Legitimacy Audit: no [ASSUMED]/[SUS]/[SLOP]). No blocking-human gate required. |
</threat_model>

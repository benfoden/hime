<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker push → panel | worker-sent ImageEntry (containing provider-sourced text) enters the chrome-extension:// panel page |
| storage.session → panel | durable entries read on open are rendered |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-16 | Tampering (XSS) | rendering pushed/stored entries | mitigate | All rendering goes through `panel-render.ts` `el()` (textContent-only, proven in Plan 04); sidepanel.ts never sets innerHTML. |
| T-12-17 | Spoofing (silent failure) | comms/listener failure | mitigate | try/catch around the message round-trip renders an explicit error state (search.ts:233-242 pattern); rebuild-from-storage.session on open means a worker restart shows persisted results, never a blank (Pitfall 5 / IMG-05). |
| T-12-18 | Information disclosure | panel must not handle the key | accept | The panel only reads `getSettings` for the target-language display name and renders entries; the googleApiKey is never sent to or read by the panel (IMG-07 enforced worker-side in Plan 05). |
</threat_model>

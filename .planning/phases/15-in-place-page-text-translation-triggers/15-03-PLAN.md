<objective>
Implement the core in-page translate→apply→toggle flow in `content.ts`. Mirror the `page-walk.ts` decision predicates verbatim (classic-script law — cannot import). On a `translatePage` message (from Plan 02's right-click/popup), do a single static `createTreeWalker` snapshot of visible, translatable Text nodes; map node→key; chunk by char budget under a concurrency cap; dispatch each chunk as `translatePageBatch` to the worker (Plan 02); write each returned translation directly into the node's `nodeValue` (layout/links/listeners untouched, no innerHTML). Keep a `WeakMap<Text,{original,translated}>` plus a strong `Text[]` so a floating corner pill (and a `togglePage` message) flips original↔translation with no reload. Mirror the page state into `chrome.storage.session` so the popup label (Plan 02) stays in sync.

Purpose: Delivers PAGE-01 (in-place replace, layout preserved), PAGE-02 (skip-set + interactivity), PAGE-03 (toggle/re-apply no reload), and PAGE-05 (static snapshot). <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Page DOM text → content snapshot | Author-controlled text read as `nodeValue` strings only (no eval, no HTML parse) |
| LLM reply (via worker) → in-place write | Translated strings written as plain text into `Text.nodeValue` |
| content → worker (translatePageBatch) | Source text crosses to the worker; the BYOK key never crosses toward the page |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-07 | Tampering / EoP (XSS) | in-place replace | mitigate | Write ONLY `Text.nodeValue` (Pattern 3) — never `innerHTML`. LLM output is never interpreted as HTML. Grep gate: content.ts innerHTML count does not increase. Codebase textContent-only law T-13-06. |
| T-15-08 | Information disclosure | translatePageBatch payload from content | mitigate | Payload carries source text + config only; no key field. Worker reads key from storage (Plan 02). Verified Task 3 step 6 (no key in payload). |
| T-15-09 | Tampering | `document.documentElement.lang` / page text feeding code paths | mitigate | Page text used only as translation input + plain-text output; lang gate is pure string compare (banner is Plan 04). No injection surface. |
| T-15-10 | DoS (self) | layout thrash on large pages | mitigate | Read-then-write two-phase (Pitfall 5): collect + visibility-check before any nodeValue write; never interleave. Cursor-driven dispatch caps in-flight chunks at PAGE_CONCURRENCY_CAP with no busy-spin. |
</threat_model>

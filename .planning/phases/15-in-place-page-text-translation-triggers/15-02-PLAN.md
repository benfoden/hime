<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| content/popup → worker (translatePageBatch) | Page-supplied source text + config crosses into the worker; the BYOK key must never cross back toward the page |
| LLM reply → worker parse | Untrusted model reply parsed by `parsePageBatchReply` (key-injection guard) |
| worker/popup → arbitrary tab (sendMessage) | Dispatching to a tab that may be restricted/content-script-less |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | Information disclosure | `translatePageBatch` case | mitigate | API key read ONLY from `s.apiKeys[s.provider]` in the worker; never accepted from the message and never echoed in any response. Acceptance grep: no `apiKey`/`key` read from `message`/`payload`. |
| T-15-05 | Tampering | LLM reply parse in worker | mitigate | `parsePageBatchReply` (Plan 01) iterates inputKeys only, string-typed — reused here verbatim. |
| T-15-06 | Denial of Service (self) | `chrome.tabs.sendMessage` to restricted tab | mitigate | `.catch(() => {})` / try/catch + `lastError` guard on every send (right-click + popup) — Pitfall 4. Verified in Task 4 step 4. |
| T-15-15 | Spoofing | global STORAGE_PAGE_STATE session mirror read by popup | mitigate | Mirror is one global record; popup origin-checks the stored `origin` against the active tab before relabeling, so a never-translated tab never shows a stale "Show original". |
| T-15-SC | Tampering | npm/pip/cargo installs | accept | N/A — phase installs no packages (locked: no new OSS dependency); RESEARCH Package Legitimacy Audit marks this N/A. No slopcheck needed. |
</threat_model>

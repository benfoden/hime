<objective>
Add the auto-offer trigger and the partial-failure resilience to `content.ts`. On `document_end`, mirror the existing progressive boot gate (content.ts:1419-1432): read himeSettings + `document.documentElement.lang`, and when `!progShouldGateByLanguage(pageLang, target)` (page language differs) AND the origin is not in the `chrome.storage.session` dismissed set, render a slim dismissible top banner offering "Translate this page" (its button runs the same path as the `translatePage` message). Dismiss (✕) appends the origin to the session dismissed set so it stays gone for the session; the manual trigger still works. For partial failures (D-04): consume Plan 03's shared `pageFailedNodes` Set — add the whole-chunk `{error}` population path, apply all successful keys, leave failed regions in source, show ONE dismissible error toast + the red error badge (reuse `setBadge`/`badgeForKind`), and offer "Retry failed sections" that re-batches only `[...pageFailedNodes]` via `pageTranslate(...)`.

Purpose: Delivers TRIG-02 (auto-offer gated by the language gate, no spend on same-language pages), TRIG-03 (unobtrusive + dismissible, manual always available), and completes PAGE-04's resilience contract (batched pipeline with graceful partial failure). Depends on Plan 03's `pageTranslate`/`pageApplyChunk` plumbing AND its shared `pageFailedNodes` surface, plus Plan 01's `STORAGE_BANNER_DISMISSED` const.

Output: edits to `src/content.ts` only.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `<html lang>` (author-controlled) → gate | Used only in a pure string comparison (progShouldGateByLanguage); no eval/injection surface |
| origin string → session dismissed set | `location.origin` stored/compared as a plain string in chrome.storage.session |
| LLM chunk reply → failed-node classification | Untrusted reply classified success/fail; parse guard lives in the worker (Plan 01/02) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-11 | Tampering | `document.documentElement.lang` feeding the banner gate | mitigate | Used only in `progShouldGateByLanguage` pure string compare (T-14-04 precedent); fail-safe direction is gate-ON (no banner, no spend). |
| T-15-12 | EoP (XSS) | banner + toast DOM | mitigate | textContent-only, no innerHTML; idempotent-by-id; mirror progCreateIndicator (T-13-06). Grep gate: content.ts innerHTML count unchanged. |
| T-15-13 | Spoofing | per-origin dismissal keyed on location.origin | accept | Origin is the correct trust granularity for "this site" (A7); a same-origin page legitimately shares the dismissal. storage.session is per-profile + ephemeral — no cross-origin leakage. |
| T-15-14 | DoS (self) / cost | retry-failed loop | mitigate | Retry re-batches ONLY `[...pageFailedNodes]` (not the whole page); success removes nodes; repeated failure re-fires one (singleton) toast, no runaway loop (each retry is user-initiated). |
</threat_model>

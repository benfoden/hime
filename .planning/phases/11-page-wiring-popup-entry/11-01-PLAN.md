<objective>
Extend the worker-side `searchTranslated` contract so the query itself is translated (explicit source→target) before the Brave search, and the translated query string is returned to the page. This is the worker half of SRCH-02 and the data source for the disclosure line (D-04/D-05).

Purpose: Today `searchTranslated` Brave-searches the query untranslated and returns only `{ results, direct }`. The page cannot show a disclosure line or run a real translated search until the worker (a) translates the query with an explicit direction (D-02), (b) short-circuits when source==target (D-03), (c) returns `translatedQuery` (D-04), and (d) falls back to the raw query on LLM failure (D-10).

Output: A pure `buildQueryTranslateConfig` helper (testable), an extended `SearchTranslatedResponse` type, and an updated `searchTranslated` handler that performs the query-translate step. No page changes — Plan 03 consumes this.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page → worker (searchTranslated message) | Untrusted query string + language names cross from the search page into the worker |
| worker → LLM provider | Query text leaves to the configured LLM endpoint over HTTPS host_permissions |
| worker → Brave API | Translated/raw query leaves to Brave over HTTPS host_permissions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Information disclosure | Brave + LLM API keys | mitigate | Keys read from chrome.storage in the worker only (`settings.braveApiKey`, `settings.apiKeys[provider]`); never accepted from message payload (XLT-01 / T-08-07). Acceptance grep enforces no payload-sourced keys. |
| T-11-02 | Tampering | LLM batch/query reply | mitigate | Translated query is a single string assigned to `searchQuery`/`translatedQuery` and `.trim()`-ed; it is searched and later rendered via textContent only (renderSerp + disclosure plan 03). No eval, no innerHTML, no key-injection (single value, not keyed JSON). |
| T-11-03 | Denial of service | LLM query translation hang | mitigate | 8s Promise.race timeout (reused from translateBatch) → AbortError → raw-query fallback (D-10), so a hung LLM never blocks results. |
| T-11-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies introduced by this plan (pure helper + handler edit). No install step. |
</threat_model>

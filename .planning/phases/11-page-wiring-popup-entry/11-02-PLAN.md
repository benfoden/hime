<objective>
Add a "Search" entry point to the toolbar popup that opens the bundled Translated Search page in a new tab with an empty search box (SRCH-01 / D-08).

Purpose: Today the popup has no path to search.html — the search feature is unreachable from the UI. D-08 keeps the popup a pure launcher: a single button that opens search.html via `chrome.tabs.create(chrome.runtime.getURL('search.html'))`, with NO query input and NO pre-fill (the deferred `?q=` deep-link is explicitly out of scope).

Output: A new button in popup.html and its click handler in popup.ts. Fully independent of the worker/page plans — runs in Wave 1 in parallel with Plan 01.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04 | Tampering | Tab URL construction | mitigate | URL is `chrome.runtime.getURL('search.html')` with a hardcoded literal path — no user/query string concatenated into the URL (D-08 keeps the popup pre-fill-free), eliminating open-redirect / param-injection surface. |
| T-11-05 | Elevation of privilege | popup permissions | accept | `chrome.tabs.create` for an extension-own page requires no extra host permission and opens only a bundled resource; low risk. |
| T-11-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies introduced by this plan. No install step. |

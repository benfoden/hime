<objective>
Build the pure, node-testable cost-guard and dedup core for progressive mode: the
≥150px eligibility filter, the per-page budget (counts STARTED jobs, D-02a), the
concurrency cap (2), the content-hash dedup key (PROG-03), and the dwell-debounce
scheduler (D-01). This module holds NO chrome.* and NO document references so it is
unit-tested against `dist/` exactly like `image-resolve.ts` — the content script
(Plan 03) and the worker (Plan 04) import the SAME logic, so the guards are proven
once here rather than re-derived in two browser-only files.

Purpose: Cost control (PROG-04) and re-bill prevention (PROG-03) are the hard
prerequisites of progressive mode — getting them wrong bills the user per scroll.
Proving them as pure functions first is the only way to verify them (project law:
never assert via the service-worker console).

Output: `src/progressive-guard.ts` (pure logic), `src/progressive-mock.ts`
(fixtures), `test/progressive-guard.mjs` (RED-first spec).
</objective>

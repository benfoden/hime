<objective>
Build the progressive viewport engine inside the content script: scan eligible
`<img>` elements (≥150px long edge), observe them with an IntersectionObserver
(rootMargin fire-ahead), apply a dwell debounce so flyby scrolling does not bill,
enforce the per-page budget and concurrency cap content-side, dedup by content key
so re-scroll never re-bills, and on each result render a small on-image badge plus
a persistent "progressive ON" indicator — ALL via textContent-only DOM (page
rendering law). Clicking a badge is the one sanctioned user gesture that asks the
worker to open the side panel at that image (D-04); the observer itself NEVER opens
the panel (PROG-06). The whole engine starts/stops live from the settings toggle
via storage.onChanged (PROG-01 no-reload).

Purpose: This is the user-facing half of progressive mode (PROG-02/04/05/06). It
reuses the proven cost-guard logic (Plan 01) and the single worker funnel (Plan 12
translateImage) — it adds NO new translation path.

Output: progressive engine appended to `src/content.ts`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page DOM → content script | Page-controlled `<img>` srcs and dimensions drive triggering; attacker page could try to inflate cost or inject via rendered strings |
| content script → worker | Only {srcUrl, dedupKey, tabId?}-shaped payloads cross; BYOK keys never leave the worker |
| observer (non-gesture) → sidePanel | Chrome forbids sidePanel.open outside a gesture; the observer must never attempt it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-05 | Denial of Service (cost) | a media-heavy/hostile page fanning out paid calls | mitigate | Eligibility(≥150px) + dwell debounce + concurrency(2) + per-page budget(10, counts starts) all enforced content-side; budget exhaustion disconnects the observer (PROG-04). |
| T-13-06 | Tampering/XSS | on-image badge + indicator rendered in page context | mitigate | textContent-only DOM build (no innerHTML) per Phase 12 page rendering law; CI grep gate forbids innerHTML in the progressive UI. |
| T-13-07 | Elevation/Info | sidePanel.open from a non-gesture context | mitigate | Observer never calls open/sends openImagePanel; ONLY the human badge-click does (PROG-06 / D-04). Verified by checkpoint step 7. |

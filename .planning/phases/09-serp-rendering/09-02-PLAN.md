<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `?state=` URL param → mock selection | `location.search` is read in `search.ts`; an unrecognized key must safely fall back to the populated default (no crash, no injection — the param only selects a fixed in-repo mock, never constructs DOM). |
| Brave-derived data → DOM | Inherited from Plan 01 — `search.ts` passes the same `SerpState` to the same `renderSerp`; this page does not introduce any new HTML-injection path. |
| Extension page → MV3 CSP | `search.html` must use an external module script only; inline script/handlers are forbidden by MV3 extension-page CSP. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Tampering | `search.ts` `?state=` param handling | mitigate | The param only indexes a fixed `MOCKS` record; unrecognized/absent keys fall back to `DEFAULT_STATE` (populated). No DOM is built from the raw param value — it is never passed to `textContent`/`createElement`. Dev-only driver, replaced by the worker in Phase 11. |
| T-09-05 | Elevation | `search.html` inline script / event handlers (MV3 CSP) | mitigate | External `<script type="module" src="search.js">` only; no inline `<script>` body and no inline `on*` handlers (repo convention, RESEARCH Security Domain). Acceptance criteria grep guards this. |
| T-09-01 (inherited) | Tampering / Elevation | renderSerp (Brave description/title) | mitigate (HIGH) | Inherited from Plan 01 — this page reuses the same textContent-only renderer; the Task 3 human check corroborates the XSS probe is inert in a real browser. No new injection surface added here. |
| T-09-02 (inherited) | Tampering | result anchor href | mitigate | Inherited — verbatim href + `rel="noopener noreferrer"` come from renderSerp (Plan 01); this page changes nothing about href construction. |
</threat_model>

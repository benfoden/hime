    - `errorNotice(doc, state)` — a `.serp-error` notice whose textContent is `state.message`; you MAY also set a `data-error-kind` attribute to `state.errorKind` so copy is distinguishable per kind. NO retry/setTimeout/setInterval anywhere (SERP-05 — quota does not auto-retry).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Brave-derived data → DOM | `SearchResult.description`/`title`/`hostname` originate from the Brave API (untrusted HTML) and cross into the rendered DOM tree. This is the only meaningful trust boundary in Phase 9. |
| Brave URL → anchor href | `SearchResult.url` crosses into an `<a href>` that the user can click to leave the extension. |
| npm registry → dev toolchain | `linkedom` install crosses into the developer's environment (test-only, never shipped). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering / Elevation | `serp-render.ts` snippet/title rendering (Brave `description`, `title`) | mitigate (HIGH — blocks phase) | V5 Output Encoding: render every Brave-derived string via `textContent`/`createElement` only; NEVER `innerHTML`/`DOMParser`-into-DOM. `test/serp.mjs` asserts `querySelectorAll('script').length === 0` and snippet `.children.length === 0` after the `<script>alert(1)</script>` probe (SERP-03). This is the dominant control of the phase. |
| T-09-02 | Tampering | `resultRow` anchor href (open-redirect / URL dressing) | mitigate | `a.href = r.url` verbatim — no `new URL()`, no `encodeURI`, no `.replace`; `rel="noopener noreferrer"` on the external anchor. Test asserts `getAttribute('href')` equals mock url byte-for-byte (SERP-02). |
| T-09-03 | Information Disclosure | `faviconEl` (per-row host leak to a third party) | mitigate | D-01: no external favicon service. Use `faviconUrl` only if Brave already provided it; otherwise a deterministic CSS letter-tile — zero per-row network, no new host permission. Test asserts no-faviconUrl rows render a `.serp-tile` span, not an img. |
| T-09-SC | Tampering | `npm install linkedom` ([ASSUMED] package, slopcheck unavailable) | mitigate | Blocking human-verify checkpoint (Task 1) before install — confirm provenance on npmjs.com (author WebReflection, ~2.7M dl/wk, no postinstall). Test-only dep, never imported by `src/` so never reaches `dist/`. |
</threat_model>

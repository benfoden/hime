## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-09 | Information Disclosure | navigator.clipboard.writeText of OCR/translation text | accept | The copied string is the exact text the user is already viewing in the panel and explicitly clicked Copy to obtain — a user-initiated, intentional copy. No new exposure; clipboard write requires the user's click gesture. |
| T-14-10 | Tampering | data-copy attribute value rendered/read back | mitigate | The value is read via `getAttribute` (a plain string) and passed to `writeText` — never parsed as HTML, never assigned via innerHTML. Feedback labels use textContent. No injection path. |
| T-14-11 | Denial of Service | rapid repeated Copy clicks / timeout races | accept | The feedback timeout is guarded (single pending timer per button); worst case is a flickering label, no security impact. |
| T-14-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan; no new dependency surface. |
</threat_model>

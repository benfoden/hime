<objective>
Add the D-05 page-language gate: progressive auto-translation must fire only when the page's `<html lang>` differs from the user's translation target, and must default OFF when `lang` is missing or ambiguous — so a page already in the user's reading language never wastes paid Vision/LLM calls. The comparison is a pure, node-testable predicate in `progressive-guard.ts`; the `<html lang>` DOM read and the gate placement live in `content.ts`. This plan also lands the D-04 on-image badge label change (`[hime]` → `[hime N]`) since both edits live in `content.ts` and the right-click path stays untouched (D-05b).

Purpose: D-05 is a new sibling cost-guard — conservative-by-default, free (no API call, just `document.documentElement.lang`). It is the cheapest possible spend reduction and must never gate the explicit right-click gesture.
Output: Pure `shouldGateByLanguage` in `progressive-guard.ts` with node tests; a gate check before `startProgressive` (both the boot path and the live-toggle path) in `content.ts`; the on-image badge rendering `[hime N]` from the extended progressiveBadge payload.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page DOM → content script | `document.documentElement.lang` is attacker/author-controlled page input feeding the D-05 gate. |
| worker → content (progressiveBadge) | `himeNum` from the worker is rendered into the on-image badge. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-04 | Tampering | `<html lang>` feeding shouldGateByLanguage | mitigate | The lang string is ONLY used in a pure base-subtag string comparison — never eval'd, never concatenated into a fetch URL, never assigned as HTML. A hostile `lang` value can at most flip the gate (fail-safe direction is gate-ON/skip, which spends nothing). No injection surface. |
| T-14-05 | Spoofing | hostile page sets lang to mismatch target to force auto-spend | accept | Even a forced gate-OFF still passes through every existing PROG cost guard (per-page budget 10, concurrency 2, min-size, dwell) and requires `progressiveEnabled === true` (explicit opt-in + first-enable consent). The page cannot enable progressive mode; worst case is the same bounded spend the user already consented to. |
| T-14-06 | Information Disclosure | himeNum in on-image badge | accept | A monotonic integer; reveals nothing sensitive. Rendered via textContent (T-13-06), no DOM injection. |
| T-14-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan; no new dependency surface. |
</threat_model>

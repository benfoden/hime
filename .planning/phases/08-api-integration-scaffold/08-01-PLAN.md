<objective>
Lock the shared type contract and error model for the entire Translated Search feature. This plan
adds the full `SearchResult` interface (consumed by Phase 9's renderer), the `searchTranslated` /
`testBraveKey` message types, the top-level `Settings.braveApiKey` field with a migration default,
and a Brave-specific error classifier that maps HTTP 429 to a distinct `search_quota` kind reading
"search quota exceeded". No network or UI code here — this is the foundation every later plan in the
phase imports.

Purpose: Phase 9 declares a hard dependency on the `SearchResult` contract being established here
(D-01). The error taxonomy must distinguish Brave quota (429) from LLM rate-limiting before any
handler can classify failures (D-07). Both must exist before BraveSearchClient (Plan 02) compiles.

Output: Extended `src/types.ts`, extended `src/errors.ts`, new test cases in `test/unit.mjs`.
</objective>

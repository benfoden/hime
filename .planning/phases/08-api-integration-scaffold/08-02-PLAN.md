<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| background worker → api.search.brave.com | the only outbound network call; carries the user's Brave key in the X-Subscription-Token header over HTTPS |
| Brave response → SearchResult | untrusted response JSON; description carried raw (HTML), url carried verbatim |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | Information disclosure | apiKey in logs | mitigate | No console.log of the key, header object, or URL with token; grep gate asserts zero console.log in brave-search.ts |
| T-08-04 | Tampering | q query param | mitigate | URL built only with `url.searchParams.set()` — encodes automatically; no manual string interpolation of user input |
| T-08-05 | Denial of resource | 429 handling | mitigate | No auto-retry on 429 (D-07); single AbortController timeout (15s) bounds hung requests |
| T-08-06 | Spoofing / SSRF | endpoint URL | accept | Endpoint is a hardcoded BRAVE_ENDPOINT constant — no user-controlled URL, no SSRF surface (per phase security note) |
| T-08-SC | Tampering | npm installs | n/a | No package installs (native fetch/URL/AbortController only — RESEARCH Package Legitimacy Audit: none) |
</threat_model>
